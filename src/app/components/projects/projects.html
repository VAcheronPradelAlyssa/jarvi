Je sais que vous avez dit : « Attention, ce poste n’est probablement pas fait pour vous si : ❌ Vous êtes nettement plus front que back (vous préférez l’UI, le CSS que la manipulation de données) »

Mes projets persos présentés ici sont effectivement 100% front (Angular, TypeScript, SCSS, HTML), car ce sont ceux qui sont utilisés par d’autres aujourd’hui.

Mais honnêtement, je trouve le travail sur le back bien plus stimulant et prenant en entreprise : c’est là que je m’éclate le plus, que je réfléchis, que je résous des vrais problèmes et que je me sens utile !

En alternance, j’ai surtout travaillé côté back : définition d’entités en XML, migration et enrichissement de données, utilisation de repositories pour accéder et manipuler les données de façon sécurisée, intégration d’API externes (ex : Argus pour l’automobile), logique métier, gestion des droits, reporting…
(Exemple : pour l’API Argus, j’ai conçu des méthodes Java pour sauvegarder, enrichir et synchroniser toutes les infos véhicules dans notre base, en veillant à la cohérence, la sécurité et la robustesse des traitements. J’ai aussi travaillé sur la génération de rapports et la validation des accès côté contrôleur.)

Mes réalisations personnelles

Site de l’AAPPMA Brochet Basse Vilaine

Contexte :
Lorsqu’on m’a confié la reprise du site après sa suppression totale et soudaine, j’ai dû tout reconstruire d’urgence, sans aucune base ni information disponible. J’ai donc recréé le site pour que l’association retrouve rapidement sa présence en ligne et puisse à nouveau informer ses membres.

Je suis devenu bénévole à l’AAPPMA en 2023, puis secrétaire de l’association en 2024.

  • Refonte complète sous Angular + TypeScript, HTML et SCSS, pour obtenir un site fiable, clair et facile à maintenir.
  • Périmètre fonctionnel volontairement simple : tout le contenu (pages, infos, agenda, documents…) est géré à la main dans le code ; il n’y a pas d’administration, de formulaire dynamique ni de base de données côté serveur.
  • Orientation utilisateur : le but principal était de remettre à disposition toutes les informations essentielles de façon accessible et responsive, même pour un public peu à l’aise avec l’informatique.
  • Gestion technique : je m'occupe de tout : ajout/mise à jour du contenu, design, développement et mise en ligne. Toute modification d’infos demande une modification dans le code source.
🌐 Découvrir le site Code sur GitHub

Prochaines étapes / Améliorations envisageables

  • Ajout d'une base de données & interface d'administration :
    - Mise en place d’un back-office sécurisé (accès selon rôle) pour permettre aux bénévoles d’ajouter, modifier ou archiver les articles/news, les événements du calendrier, etc, sans passer par le code.
    - Utilisation possible : stack Java (Spring Boot) côté serveur avec une base PostgreSQL.
  • Publication d’annonces, actualités et documents en autonomie :
    - Interface simple de rédaction et de gestion des ressources (images, PDF, etc.), avec système de prévisualisation avant publication.
  • API publique & connecteurs externes :
    - Exposition d’API REST sécurisées pour intégrer, par exemple, les calendriers d’événements sur d’autres plateformes, synchroniser des alertes météo/pêche automatisées, ou interfacer des systèmes de paiement pour les adhésions en ligne.
  • Accessibilité accrue et SEO optimisé :
    - Amélioration des contrastes, navigation clavier, lecture facile pour les malvoyants.
    - Sémantique HTML optimisée, balises OpenGraph, sitemap.xml, meta pour améliorer la visibilité du site sur les moteurs de recherche.
  • Formulaires dynamiques (contact, inscriptions aux événements, suggestions)...
    - Validation côté client/serveur, gestion RGPD basique (consentement, suppression de données sur demande), confirmation email automatique lors d’inscription.
  • Maintenance facilitée et monitoring :
    - Intégration d’outils de supervision (statistiques de fréquentation, alertes en cas de problème serveur, monitoring automatique des pages clés du site).
  • Hébergement des médias hors du code source :
    - Stockage des photos, documents et ressources sur un espace cloud dédié (type Azure Blob, AWS S3, Google Cloud Storage…) pour éviter d’alourdir le dépôt Git et faciliter la gestion/ajout de médias sans redéployer le site.
Avec ces évolutions, l’association disposerait d’un outil à la fois robuste, autonome et évolutif, capable de s’adapter à tous les besoins futurs : communication, gestion, vie communautaire et valorisation de ses activités… sans dépendre d’un unique développeur.
Ce projet m’a surtout permis de prouver ma réactivité, ma capacité à livrer dans l’urgence, et mon souci du rendu pour des utilisateurs non-tech. Même si la partie gestion de données est très limitée ici, l’expérience m’a motivée à aller bien plus loin (cf. mes autres projets à venir, pensés fullstack).

Ramène Ta Laine (en cours)

Pitch : Plateforme communautaire d’échanges, de ventes et d’achats de pelotes de laine, de patrons numériques et de matériel de tricot/crochet entre particuliers. Objectif : centraliser l’offre, simplifier la recherche, permettre des transactions sécurisées et encourager la création autour du tricot & du crochet.

  • Back-end : Java Spring Boot (REST API), gestion des utilisateurs, annonces, commandes, messagerie, paiement Stripe, publication d’annonces/patrons… (base PostgreSQL ou équivalent).
  • Front-end : Application Angular responsive pour une UX “communauté”, accessible sur mobile et desktop.
  • Fonctionnalités prévues :
    • Inscription/connexion sécurisée (JWT)
    • Création et gestion d’annonces, mise en vente et achat de pelotes
    • Publication, vente & achat de patrons numériques PDF (bibliothèque personnelle et téléchargement)
    • Paiement intégré (Stripe)
    • Messagerie interne membres
    • Notifications, favoris, historique
    • Recherche avancée (filtres laine, patrons, catégorie…)
  • État d’avancement : Cahier des charges complet, premières maquettes visuelles et modèles de données établis, début du développement Angular.
Prochaines étapes : Développement technique courant 2026 : mise en place des endpoints, gestion sécurité, administration, puis front complet.
L’objectif est d’offrir, enfin, à la communauté une alternative saine, indépendante et pérenne aux groupes Facebook pour les échanges tricot/crochet/patrons.

Independant'fish (en réflexion) projet personnel, objectif fin 2026

Pitch : Marketplace de pêche nouvelle génération : mise en avant des artisans créateurs (leurres, plombs, accessoires…) et vente de matériel d’occasion spécialisé pour passionnés de pêche. Le but : rassembler professionnels/artisans indépendants et communauté pêche autour d’une plateforme unique.

  • Back-end : Java Spring Boot (API REST), gestion utilisateurs, boutiques, annonces, commandes, messagerie, paiements, système d’avis et forum. Base PostgreSQL/Supabase prévue.
  • Front-end : Angular (web app responsive, mobile-first), navigation rapide et mise en avant visuelle des artisans chaque semaine.
  • Fonctionnalités annoncées :
    • Mises en avant d'un artisan chaque semaine sur la home
    • Création de boutiques artisans, gestion produits/vitrine, avis clients
    • Vente de matériel de pêche d’occasion, filtrable par type, prix, état, localisation
    • Messagerie & commandes intégrées
    • Forum / Q&R communautaire (évolution future)
    • Système d’abonnement info/newsletter, contacts client/artisan
  • Maquettage & outils : Premières maquettes réalisées sur Visily (apprentissage prototypage UI/UX)
Prochaines étapes : Affinage du cahier des charges, structuration de la base, puis développement technique progressif courant 2026.
